## Component Details

### Open WebUI
**Role:** Primary user-facing chat interface.
**Container image:** `ghcr.io/open-webui/open-webui:v0.8.10`
**Internal port:** 8080 → host `127.0.0.1:9090`
**Traefik hostname:** `openwebui.<domain>`
**Auth:** Authentik forward auth; internal user accounts
**Key features:**
- Multi-user with role-based access (admin / user)
- OpenAI-compatible API pass-through to LiteLLM
- Model switching; per-conversation model selection
- RAG pipeline: upload documents, web search integration
- Tool/function calling support
- Image generation (via Stable Diffusion or DALL-E backends)
- Mobile-responsive; PWA-installable
- OIDC/OAuth2 login via Authentik
- Plugin/extension system

---

### Flowise
**Role:** Visual no-code LLM agent and workflow builder.
**Container image:** `docker.io/flowiseai/flowise:3.0.13`
**Internal port:** 3000 → host `127.0.0.1:3001`
**Traefik hostname:** `flowise.<domain>`
**Auth:** Authentik forward auth (proxy-only — no OIDC integration)
**Key features:**
- Drag-and-drop pipeline builder (LangChain-based)
- Multi-step RAG chains with vector store integration
- Agent loops with tool calling
- Credentials vault for API keys
- REST API deployment for built flows
- Memory / conversation history nodes
- Webhook triggers

---

### Authentik
**Role:** Identity provider, single sign-on gateway for all services.
**Container image:** `ghcr.io/goauthentik/server:2026.2.1`
**Internal port:** 9000 (HTTP), 9443 (HTTPS)
**Traefik hostname:** `auth.<domain>`
**Database:** PostgreSQL
**Key features:**
- OIDC/OAuth2 provider (used by Grafana, Open WebUI)
- SAML 2.0 support
- LDAP outpost (optional)
- Forward authentication for Traefik (used by Flowise, Prometheus, LiteLLM admin)
- User self-service portal; admin UI
- Multi-factor authentication (TOTP, WebAuthn)
- Application catalog and access control policies
- Social login providers (GitHub, Google, etc.)
- Community edition: all features, MIT license; Enterprise adds SLA/support

---

### LiteLLM
**Role:** OpenAI-compatible model routing proxy; single endpoint for all inference backends.
**Container image:** `ghcr.io/berriai/litellm:main-v1.81.14-stable`
**Internal port:** 4000 → host `127.0.0.1:9000`
**Traefik hostname:** `litellm.<domain>`
**Config source:** `configs/models.json` (generated by `configure.sh generate-litellm-config`)
**Key features:**
- OpenAI-compatible `/v1/chat/completions`, `/v1/models`, `/v1/embeddings`
- 100+ provider support (Ollama, vLLM, OpenAI, Anthropic, Groq, Azure, Bedrock, …)
- Load balancing across multiple instances of the same model
- Virtual API keys with per-key spend limits and rate limiting
- Cost tracking and usage logging
- Per-model fallback and retry logic
- Health check endpoints per backend
- Admin UI (Prometheus metrics, key management)
- Prompt caching integration

---

### vLLM
**Role:** High-throughput GPU inference server for locally-hosted models.
**Container image:** `docker.io/vllm/vllm-openai:v0.17.1`
**Internal port:** 8000
**Device:** RTX 3070 Ti (8 GB VRAM) via CDI (`nvidia.com/gpu=all`)
**Current model:** `Qwen/Qwen2.5-1.5B-Instruct` (FP16, served as `qwen2.5-1.5b`)
**Key features:**
- PagedAttention for near-linear VRAM scaling with batch size
- OpenAI-compatible API (`/v1/chat/completions`, `/v1/completions`)
- Quantization support: AWQ, GPTQ, INT4, INT8, FP8
- Tensor parallelism for multi-GPU
- Streaming completions
- Continuous batching (high throughput under concurrent load)
- LoRA adapter loading
- Speculative decoding

---

### Ollama
**Role:** Local model manager for CPU inference; used by Knowledge Index for embeddings; target runtime for remote inference workers.
**Container image:** `docker.io/ollama/ollama:0.17.7`
**Internal DNS:** `ollama.ai-stack:11434`
**Device:** CPU-only on controller (`CUDA_VISIBLE_DEVICES=""`)
**Key features:**
- Model download and lifecycle management (GGUF format)
- OpenAI-compatible API
- Embedding endpoint (`/api/embeddings`)
- Modelfile customization (system prompts, parameters)
- GPU acceleration: CUDA (Linux), Metal (macOS), ROCm (AMD)
- Streaming completions
- Concurrent request handling
- Supports quantized models (Q4_K_M, Q8_0, etc.) natively

---

### Knowledge Index Service *(custom)*
**Role:** RAG backend providing document ingestion, vector search, and MCP tool interface.
**Source:** `services/knowledge-index/`
**Container image:** `localhost/knowledge-index:0.1.0` (built locally)
**Internal port:** 8100 → host `127.0.0.1:8100`
**Dependencies:** Ollama (embeddings), Qdrant (vector storage)
**Key features:**
- REST API: `POST /documents` (ingest), `PUT /documents/{id}` (update), `DELETE /documents/{id}`, `POST /query` (vector search), `GET /health`
- MCP HTTP/SSE transport: `GET /mcp/sse`, `POST /mcp/messages`
- MCP tools: `ingest_document`, `search_knowledge`
- Chunking pipeline (sentence-aware, configurable `CHUNK_SIZE`)
- Ollama-backed embedding (`EMBED_MODEL` env var)
- Qdrant-backed multi-collection vector storage
- API key auth (`API_KEY` env var)
- FastAPI / uvicorn runtime; Python 3.12

---

### Qdrant
**Role:** Vector database for semantic search and RAG retrieval.
**Container image:** `docker.io/qdrant/qdrant:v1.17.0`
**Internal port:** 6333 (REST) / 6334 (gRPC) → host `127.0.0.1`
**Internal DNS:** `qdrant.ai-stack:6333`
**Key features:**
- High-performance approximate nearest-neighbor search
- Named collections with configurable distance metrics (Cosine, Euclid, Dot)
- Payload filtering (structured metadata alongside vectors)
- gRPC and REST API
- On-disk HNSW indexing
- Scalar and product quantization for memory reduction
- Snapshot/backup API
- Multi-tenant via payload-based access patterns
- Community (Apache-2.0) and Cloud/Enterprise offerings

---

### PostgreSQL
**Role:** Relational database backend for Authentik (identity data) and Flowise (flow/credential storage).
**Container image:** `docker.io/library/postgres:17.9`
**Internal port:** 5432 → host `127.0.0.1:5432`
**Internal DNS:** `postgres.ai-stack:5432`
**Key features:**
- ACID-compliant relational database
- JSONB native type (schema-flexible storage)
- Full-text search
- Row-level security
- Logical replication
- `pgvector` extension available (not used in current stack; Qdrant handles vectors)
- PostgreSQL License: permissive BSD-style; no copyleft restrictions

---

### Traefik Proxy
**Role:** Reverse proxy, TLS termination, and forward-auth gateway for all user-facing services.
**Container image:** `docker.io/library/traefik:v3.6.10`
**Ports:** 80 (HTTP redirect), 443 (HTTPS), `127.0.0.1:8080` (dashboard)
**Config:** `configs/traefik/` (static `traefik.yaml` + dynamic `dynamic/`)
**Key features:**
- Automatic TLS via Let's Encrypt (ACME) or static certificates
- File-based dynamic config (hot-reload without restart)
- Forward authentication middleware (Authentik integration)
- PathPrefix and Host-based routing rules
- Rate limiting and retry middleware
- TLS mTLS support
- Dashboard and metrics endpoint
- Community (MIT, Traefik Proxy) and Enterprise editions separate

### Loki
**Role:** Log aggregation backend.
**Container image:** `docker.io/grafana/loki:3.6.7`
**Internal port:** 3100 → host `127.0.0.1:3100`
**Config:** `configs/loki/local-config.yaml`
**Key features:**
- Label-based log indexing (low-cardinality, cost-efficient vs. Elasticsearch)
- LogQL query language (grep + metric extraction)
- Filesystem and object storage backends (S3, GCS, Azure)
- Log retention policies
- Ruler for log-based alerting
- Native Grafana datasource integration
- ⚠ AGPL-3.0: same note as Grafana

---

### Promtail
**Role:** Log shipping agent — collects logs from containers and systemd, ships to Loki.
**Container image:** `docker.io/grafana/promtail:3.6.7`
**Config:** `configs/promtail/config.yml`
**Key features:**
- Tails local log files and systemd journal
- Adds labels (service name, host, container) before shipping
- Supports pipeline stages (regex extraction, JSON parsing, label transformation)
- Backfill on startup (does not lose logs during restart)
- Runs as sidecar or standalone agent
- Used on inference-worker nodes (Podman profile) to forward logs to controller Loki
- ⚠ AGPL-3.0: same note as Grafana
