# 1 Deployment Preferences

- Podman v5.7+ required; rootless mode exclusively
- Dedicated service user with lingering enabled
- All containers managed via systemd quadlets generated by `scripts/configure.sh`
- Base directory: `$AI_STACK_DIR` (default `$HOME/ai-stack`)
- GPU access via CDI: `nvidia.com/gpu=all` for vLLM

# 2 Configuration Choices

- Single Podman network: `ai-stack-net` (bridge mode, DNS enabled)
- Named volumes are not used; all data stored as bind mounts under `$AI_STACK_DIR/data/<service>/`
- Quadlet files placed in `$HOME/.config/containers/systemd/`
- Secrets provisioned via `podman secret create` from `configs/config.json` values
- Health checks defined in quadlet files matching definitions from `configs/config.json`

# 3 Operational Conventions

- Start/stop services via systemctl: `systemctl --user start ai-stack-<service>.service`
- View logs: `journalctl --user -u ai-stack-<service>.service`
- Check status: `systemctl --user status ai-stack-<service>.service`
- Validate stack health: `scripts/validate-system.sh`
- Generate/regenerate quadlets: `scripts/configure.sh generate-quadlets`

# 4 Troubleshooting Patterns

- Container won't start: `systemctl --user status ai-stack-<service>` → check ExecStart errors
- Network issues: `podman exec <container> curl http://<target>:<port>/health`
- Permission denied: `podman unshare ls -la <path>` to check UID mapping
- Image pull failures: verify registry configuration and network connectivity
- Resource exhaustion: `podman stats --no-stream` to identify over-consuming containers
