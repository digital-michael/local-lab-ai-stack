# 1 Secret Management

- `AUTHENTIK_SECRET_KEY` must be cryptographically random (minimum 50 characters) and never reused
- Inject all secrets (secret key, database password) via Podman secrets
- Client secrets for OIDC applications should be generated by Authentik, not manually set
- Rotate the secret key carefully — it invalidates existing sessions and tokens
- Store recovery keys and bootstrap tokens securely offline

# 2 Authentication Policies

- Enforce password complexity: minimum 12 characters, mixed case, numbers, special characters
- Enable multi-factor authentication (MFA) for admin accounts at minimum
- Set account lockout after 5 failed login attempts
- Configure password expiration policies appropriate to the environment
- Use Authentik's reputation system to detect and block brute-force attacks
- Disable unused authentication sources (LDAP, SAML) to reduce attack surface

# 3 Network Security

- Expose Authentik's web UI only through the TLS reverse proxy
- Internal services connect via the Podman network for OIDC token verification
- Set CORS origins precisely; do not use wildcards
- Configure `X-Forwarded-For` trust to the reverse proxy IP only
- Rate-limit login endpoints at the proxy layer

# 4 Token Security

- Use short-lived access tokens (5–15 minutes) with refresh token rotation
- Sign ID tokens with RS256 for asymmetric verification
- Set appropriate token audience claims to prevent token reuse across applications
- Invalidate refresh tokens on password change or explicit logout
- Do not expose token signing keys; only the JWKS endpoint should be public

# 5 Audit and Monitoring

- Authentik logs all authentication events by default — ship these to Loki via Promtail
- Monitor for: failed login spikes, new admin account creation, OIDC application changes
- Set alerts for authentication failures exceeding baseline thresholds
- Review audit logs regularly for unauthorized access attempts
- Export audit data for compliance if required

# 6 Container Security

- Run as a non-root user inside the container
- Use rootless Podman for user-namespace isolation
- Pin the image to a specific tag or digest
- Scan the image for CVEs; Authentik pulls Python and Django dependencies with a large surface
- Drop unnecessary capabilities
